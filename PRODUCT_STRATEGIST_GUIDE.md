# PRODUCT_STRATEGIST_GUIDE.md - Product Strategist를 위한 완벽한 가이드

## 🎯 Product Strategist의 역할

제품 비전을 수립하고, 시장을 분석하고, 로드맵을 설계하는 **전략 전문가**.

---

## 🗺️ 제품 전략 프로세스

### 1. 시장 조사 (1-2시간)
```markdown
**목적:** 경쟁 환경 이해

**질문:**
- 누가 경쟁자인가?
- 그들의 강점/약점은?
- 시장 크기는?
- 트렌드는?

**방법:**
- 경쟁사 제품 직접 사용 (Browser tool)
- 리뷰 읽기 (App Store, ProductHunt, Reddit)
- 시장 보고서 (Google Trends, Statista)
```

### 2. 사용자 리서치 (1-2시간)
```markdown
**목적:** 사용자 니즈 파악

**질문:**
- 누가 우리 사용자인가?
- 어떤 문제를 해결하려 하나?
- 현재 어떻게 해결하나?
- 우리 제품이 더 나은 이유는?

**방법:**
- 사용자 인터뷰 (5-10명)
- 설문조사 (Google Forms)
- 행동 데이터 분석 (Analyst와 협업)
```

### 3. 제품 비전 수립 (30분)
```markdown
**구조:**

**Mission (미션):**
"우리는 [누구]를 위해 [무엇]을 제공한다"

예: "우리는 퇴사를 고민하는 직장인을 위해 재정적 자신감을 제공한다"

**Vision (비전):**
"3년 후 우리는 [어떤 모습]"

예: "Personal Runway Calculator는 재정 계획의 표준 도구가 된다"

**Values (가치):**
1. 투명성: 계산 방식 공개
2. 신뢰성: 정확한 데이터
3. 사용자 중심: 간단하고 직관적
```

### 4. 로드맵 설계 (2-3시간)
```markdown
**Now (지금 - 1개월):**
- P0 기능 (MVP)
- 치명적 버그 수정
- 핵심 UX 개선

**Next (다음 - 3개월):**
- P1 기능 (중요)
- 성장 실험 (A/B 테스트)
- 플랫폼 확장 (모바일 앱?)

**Later (나중 - 6개월+):**
- P2 기능 (Nice to have)
- 새로운 시장 (B2B?)
- 혁신적 기능 (AI 조언?)
```

### 5. 우선순위 결정 (1시간)
```markdown
**프레임워크: RICE**

- **R**each: 영향 받는 사용자 수
- **I**mpact: 사용자당 임팩트 (High/Med/Low)
- **C**onfidence: 확신도 (0-100%)
- **E**ffort: 개발 시간 (person-weeks)

**RICE Score = (R × I × C) / E**

예:
- 시나리오 비교: (100 × 3 × 80%) / 1주 = 240
- 다크 모드: (200 × 1 × 50%) / 0.5주 = 200

→ 시나리오 비교 먼저!
```

## 🎯 Personal Runway Calculator 전략

### 현재 포지셔닝
```markdown
**시장:** 개인 재무 관리
**타겟:** 퇴사 준비생, 프리랜서, 창업자
**차별화:** 런웨이 중심 (vs 예산 중심)

**경쟁사:**
- YNAB: 예산 중심, 복잡함
- Mint: 자동 연동, 프라이버시 우려
- PocketSmith: 미래 예측, 비싸고 복잡

**우리 강점:**
- 간단함 (5분 내 결과)
- 런웨이 명확 (숫자 하나로 이해)
- 시나리오 비교 (의사결정 지원)
- 무료 (기본 기능)
```

### 타겟 페르소나
```markdown
**Persona 1: 퇴사 준비생 (70%)**
- 나이: 28-35세
- 직업: 직장인 (3-7년 차)
- 고민: "언제까지 버틸 수 있지?"
- 니즈: 빠른 계산, 명확한 숫자
- 감정: 불안, 혼란

**Persona 2: 프리랜서 (20%)**
- 나이: 30-40세
- 직업: 프리랜서, 계약직
- 고민: "수입 불안정한데 대비책은?"
- 니즈: 시나리오 비교, 최악 케이스
- 감정: 불안정, 조심스러움

**Persona 3: 스타트업 창업자 (10%)**
- 나이: 25-35세
- 직업: 창업자, Co-founder
- 고민: "투자 받기 전 망하면?"
- 니즈: Burn rate, 런웨이 추적
- 감정: 긴장, 압박
```

### 성장 전략
```markdown
**Phase 1: Product-Market Fit (지금)**
- 목표: 베타 사용자 100명
- 지표: NPS 50+, 리텐션 40%+
- 전략: 개인 네트워크, Reddit, ProductHunt

**Phase 2: Growth (3개월 후)**
- 목표: MAU 1,000명
- 지표: 가입 전환율 20%+, 바이럴 K=1.2
- 전략: SEO, 콘텐츠 마케팅, 입소문

**Phase 3: Scale (6개월 후)**
- 목표: MAU 10,000명
- 지표: Premium 전환 5%+, CAC < LTV
- 전략: 유료 광고, 파트너십, 플랫폼 확장
```

### 수익 모델
```markdown
**Freemium:**
- Free: 기본 기능 (1-3 scenarios)
- Premium ($5/mo): Unlimited scenarios, 고급 기능

**Premium 기능 후보:**
- 무제한 시나리오
- 고급 차트 (Projection, Monte Carlo)
- 데이터 export (CSV, PDF)
- 팀 협업 (커플, 가족)
- 재무 조언 (AI?)

**목표:**
- Year 1: 10% Premium 전환
- Year 2: MRR $10K
- Year 3: MRR $50K+
```

## 🚨 흔한 전략 실수

### ❌ 하지 말 것

1. **모든 것 다 하기**
   ```markdown
   ❌ "B2C, B2B, 모바일, 웹, AI 다 하자!"
   ✅ "먼저 B2C 웹에 집중. PMF 후 확장"
   ```

2. **경쟁사 따라하기**
   ```markdown
   ❌ "YNAB이 A 기능 있으니 우리도!"
   ✅ "우리 차별화 포인트에 집중"
   ```

3. **데이터 무시**
   ```markdown
   ❌ "내 생각엔 사용자가 이걸 원할 것 같아"
   ✅ "데이터 보니 80% 사용자가 X 기능 사용"
   ```

4. **장기 계획만**
   ```markdown
   ❌ "3년 후 우리는..." (지금 당장 뭐 할지 모름)
   ✅ "이번 주/달 목표는... 3년 비전은..."
   ```

### ✅ 반드시 할 것

1. **Focus (집중)**
   ```markdown
   한 번에 하나씩:
   - 한 타겟 세그먼트
   - 한 핵심 기능
   - 한 채널
   ```

2. **Validate (검증)**
   ```markdown
   만들기 전에:
   - 사용자 인터뷰 (최소 5명)
   - 프로토타입 테스트
   - 데이터 분석
   ```

3. **Iterate (반복)**
   ```markdown
   Build → Measure → Learn → Repeat
   - 2주 스프린트
   - 메트릭 추적
   - 배운 것 문서화
   ```

## 🎯 성공 기준

### Product Strategist가 성공한 것:
- ✅ 명확한 비전 & 미션
- ✅ 데이터 기반 로드맵
- ✅ 우선순위 명확 (RICE 등)
- ✅ 경쟁사 분석 완료
- ✅ 실행 가능한 전략
- ✅ 메트릭 정의 & 추적

### Product Strategist가 실패한 것:
- ❌ 모호한 비전 ("세상을 바꾼다")
- ❌ 주관적 로드맵 (데이터 없음)
- ❌ 우선순위 불명확 ("다 중요함")
- ❌ 경쟁사 무시
- ❌ 추상적 전략 (실행 불가)
- ❌ 메트릭 없음 (성공 판단 불가)

## 💡 Tip: 효율적인 전략 수립

### 빠른 시장 조사
```markdown
2시간 안에:
1. Google: "[카테고리] tools" → Top 10
2. ProductHunt: 최근 출시 제품
3. Reddit: r/personalfinance, r/startups
4. App Store: "budget" 검색 → 리뷰 읽기

→ 경쟁 환경 80% 파악 가능
```

### 사용자 인터뷰 템플릿
```markdown
10분 인터뷰:
1. "어떤 일 하세요?" (컨텍스트)
2. "재정 계획 어떻게 하세요?" (현재 행동)
3. "불편한 점은?" (Pain point)
4. "이상적으로는?" (Ideal solution)
5. "[우리 제품] 어떻게 생각하세요?" (피드백)

5명만 인터뷰해도 패턴 보임!
```

### 로드맵 우선순위
```markdown
의심스러우면:
1. P0 (없으면 제품 못 씀)
2. P1 (있으면 좋음, 사용자 요청 많음)
3. P2 (Nice to have, 나중에)

P0만 먼저 완성 → 출시 → 피드백 → P1
```
